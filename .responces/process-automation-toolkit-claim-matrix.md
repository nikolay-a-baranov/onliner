# Process Automation Toolkit Claim Matrix

## Safe Claims

- Built an internal browser-side workflow automation toolkit.
- Focused on editorial and CMS operations in a legacy browser/admin environment.
- Packaged repeated operational work into reusable commands and workflow modules.
- Added validation, guardrails, and manual review points for publish-sensitive flows.
- Supported structured source export and AI-assisted handoff.
- Worked with maintainability in mind through modularization and documented ownership rules.

## Safe But Better With Examples

- Improved consistency of repeated operational workflows.
- Reduced manual variation in post-preparation and admin tasks.
- Consolidated multiple repeated steps into a more usable operator tool surface.
- Turned scattered workflow helpers into a more coherent internal product layer.

## Use Carefully

- Reduced manual effort
  - Safe only if phrased without exact numbers unless measured.
- Improved productivity
  - Better replaced with concrete workflow standardization language.
- Increased reliability
  - Best tied to validation/guardrail behavior rather than broad outcomes.

## Avoid Without Extra Evidence

- Saved X hours per week
- Used by the whole team/company
- Eliminated process errors
- Built a platform
- Built a production AI system
- Owned end-to-end publishing infrastructure

## Best Replacement Phrases

- Instead of "improved productivity"
  - "standardized repeated workflow steps"
  - "reduced manual variation in recurring tasks"

- Instead of "built a platform"
  - "built an internal workflow automation toolkit"
  - "built a browser-side operator tooling layer"

- Instead of "automated everything"
  - "automated repeatable parts of the workflow while keeping operators in control"

- Instead of "AI product"
  - "AI-assisted handoff workflow"
  - "structured source export for downstream AI-assisted drafting"

## Strongest Positioning Sentence

I built an internal browser-side workflow automation toolkit that standardized repeated editorial and CMS tasks through reusable commands, operator guardrails, and structured handoff flows in a legacy admin environment.
