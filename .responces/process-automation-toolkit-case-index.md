# Process Automation Toolkit Case Index

Use this folder as a small case-study pack for job applications and interviews.

## Files

- `process-automation-toolkit-case-study-report.md`
  - Full evidence-backed report with technical and career sections.
- `process-automation-toolkit-career-pack.md`
  - Short copy-ready materials for CV, LinkedIn, portfolio, and interviews.
- `process-automation-toolkit-role-variants.md`
  - How to retell the case for different target roles.
- `process-automation-toolkit-interview-qa.md`
  - Likely interview questions with safe answers.
- `process-automation-toolkit-evidence-checklist.md`
  - What evidence is already supported and what to collect next.
- `process-automation-toolkit-claim-matrix.md`
  - Safe vs risky language for applications and interviews.

## Recommended Use Order

1. Start with `process-automation-toolkit-career-pack.md`.
2. Use `process-automation-toolkit-role-variants.md` to tailor the story to the vacancy.
3. Use `process-automation-toolkit-claim-matrix.md` to avoid overclaiming.
4. Use `process-automation-toolkit-interview-qa.md` before interviews.
5. Use `process-automation-toolkit-evidence-checklist.md` to strengthen the case over time.
6. Keep `process-automation-toolkit-case-study-report.md` as the deep proof document.

## Recommended Default Positioning

Default to:
- Process Automation Engineer
- Internal Tools Engineer

Only shift toward broader product/frontend narratives when the vacancy clearly rewards it.
