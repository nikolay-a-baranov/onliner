# Madtest Codex Pack

Цель: довести букмарклет Madtest до рабочего MVP.

## Что уже решили

- Делать один объект `madtest`, а не набор разрозненных функций.
- Запуск через `madtest.run(mode)`.
- Основные режимы:
  - `auto` — логин/роутинг.
  - `export` — экспорт текущей страницы.
  - `sanitize` — нормализация текущей страницы.
  - `find` — поиск внутренней ссылки редактирования по публичному `madte.st` id.
- Для масштабируемости страницы описываются в `madtest.pages`.
- Для разных наборов полей используется универсальный сборщик видимых редактируемых полей.
- Для `questions/results` позже может понадобиться обход элементов списка: кликнуть пункт → собрать поля справа → следующий пункт.

## Почему так

Madtest — SPA: один бандл, но DOM отличается по открытому URL. Поэтому роутинг делаем по URL, а работу с содержимым — по реально видимым полям текущей страницы.

## Команды

В консоли можно тестировать так:

```js
window.__madtestMode = "export"
```

потом запускать bookmarklet.

Или после загрузки:

```js
madtest.run("export")
madtest.run("sanitize")
madtest.run("find")
```
